호기심 : Semantic(의미론적인) HTML이란?

원론적인 이야기... ----------------------

- 구문론과 의미론 (언어론)

구문론 : 문장의 형식을 연구
I like you. - 주어 + 서술어 + 목적어
나는 너를 사랑한다. - 주어 + 목적어 + 서술어
-> 언어가 다르기 때문에 형식이 다르구나!

의미론 : 문장의 의미를 연구
I like you.
나는 너를 좋아한다.
-> 화자는 필자를 좋아하는구나! 

- 구문론과 의미론 (프로그래밍)

구문론 : 프로그램의 구조
-> 어떤 형식으로 작성해야하는가?

의미론 : 프로그램의 내용
-> 어떤 일이 일어나는가?

------------------------------------------

그래서, Semantic HTML은 뭔데?
-> Semantic elemtent로 구성된 HTML

Semantic element는 뭔데?
-> 브라우저와 개발자 모두가 의미를 알 수 있는 요소
ex) <form>, <title>...
opp ex) <div>, <span>... : non-semantic elements으로, 브라우저가 정보를 수집할 수가 없다!
-> non-Semantic element는 개발자만 해석할 수 있는 요소!
(개발자가 정보의 중요도를 파악하기 위한 도구)

왜 Semantic HTML을 만들어야하는데?
-> 검색엔진에 더 잘 노출되기 위해서.

Semantic하게 만들면 왜 잘 노출되는데?
-> 브라우저가 정보의 중요도를 파악할 수 있으므로, 검색 엔진이 개발자의 의도대로 움직여준다.
